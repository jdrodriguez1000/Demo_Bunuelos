## 1. Descripción del Cambio

Incorporar un indicador de progreso del proyecto en el Resumen Ejecutivo de cada etapa cerrada. El indicador debe ser simple para el cliente y trazable para el equipo.

**Metodología aprobada:**

**Variables dinámicas (se leen de `PROJECT_index.md` en cada cierre):**
- `N` = número total de fases del proyecto (variable, depende de cada proyecto)
- `E_i` = número total de etapas de la fase i (variable, puede crecer en cualquier momento)

**Fórmula:**
```
Peso de cada fase     = 100% / N
Avance de una fase i  = (Etapas Cerradas en i / E_i) × (100% / N)
Progreso Total        = Σ Avance de cada fase
```

**Regla de cierre:** Una etapa cuenta como cerrada **únicamente** cuando existe su archivo `docs/executives/f[F]_[E]_executive.md` en el repositorio. Sin ese archivo, no cuenta.

---

**Regla de Alcance Dinámico — Variación del Porcentaje:**

El alcance del proyecto puede crecer durante la ejecución (se agregan fases o etapas). Esto puede causar que el porcentaje de avance **baje** aunque no se haya retrocedido en el trabajo. Esto es **esperado y correcto**, y debe explicarse al cliente de forma transparente.

**Cuándo cambia el denominador:**
1. Se agrega una nueva fase → N aumenta → el peso de cada fase baja → el % total puede bajar.
2. Se agrega una etapa a una fase existente → E_i aumenta → el avance de esa fase baja → el % total puede bajar.

**Regla de comunicación obligatoria:** Cada vez que el % de progreso baje por expansión de alcance (no por trabajo revertido), el ejecutivo de la próxima etapa cerrada debe incluir una nota explicativa:

```
⚠️ Nota de Alcance: El avance bajó de X% a Y% porque se incorporaron
Z fases / W etapas nuevas al proyecto. El trabajo completado no cambió.
Este ajuste refleja un alcance más amplio, no un retroceso.
```

---

**Ejemplo A — Proyecto estable:**
```
N = 4 fases, Fase 1 tiene 3 etapas, Fase 2 tiene 4 etapas

Fase 1: 2/3 cerradas → 2/3 × 25% = 16.67%
Fases 2-4: 0% cada una

PROGRESO TOTAL = 16.67%
```

**Ejemplo B — Se agregan 2 fases (N pasa de 4 a 6):**
```
Antes: 2 etapas cerradas de Fase 1 = 2/3 × 25% = 16.67%
Después de agregar 2 fases:
  Peso por fase ahora = 100% / 6 = 16.67%
  Fase 1: 2/3 × 16.67% = 11.11%

PROGRESO TOTAL = 11.11% ← bajó de 16.67% sin retroceder trabajo
→ Se incluye nota de alcance en el próximo ejecutivo
```

## 3. Alcance e Implicaciones

Este CC establece un estándar de reporte de progreso que aplica a todas las etapas presentes y futuras del proyecto.

### Riesgos de NO hacer el cambio:
- Los clientes no tienen visibilidad del avance real del proyecto.
- Cada sesión de reporte requiere cálculo manual propenso a errores.
- El progreso no queda documentado en los ejecutivos cerrados.

### Riesgos de SÍ hacer el cambio:
- `f01_02_executive.md` es retroactivo — requiere modificar un documento de etapa cerrada.
- El skill `/close-stage` cambia su comportamiento para todas las etapas futuras.
- Al agregar fases o etapas, el % puede bajar — requiere nota explicativa al cliente para no generar confusión.
